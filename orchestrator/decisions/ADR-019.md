# ADR-019: Embeddings + Clustering + Autonomy (M4.1)

**Date:** 2026-05-28
**Status:** Accepted
**Deciders:** Cristian Molina
**Context:** Sprint M4.1 — "El Cazador" Fase 3 (aprendizaje + sugerencias)

---

## Context

Fase 3 del spec del founder pedía:
- Embeddings del contenido de las señales
- Clustering con HDBSCAN
- Scoring que aprende del feedback (up/down)
- Sugerencias de fuentes basadas en clusters aprobados
- Nivel de autonomía configurable (manual / asistido / autónomo con aprobación)

## Decision

### 1. Dependencias OPCIONALES con fallback determinista

`sentence-transformers` (80MB modelo) + `hdbscan` + `scikit-learn` son
**recomendadas pero opcionales**. El módulo funciona siempre:

- **Real mode**: si las deps están instaladas — embeddings semánticos
  multilingüe (`paraphrase-multilingual-MiniLM-L12-v2`)
- **Fallback mode** (default): embeddings hash-based determinísticos.
  Sin red, sin descarga, sin instalación. Tests verdes en CI sin instalar
  500MB de deps.

`get_engine_info()["mode"]` reporta cuál está activo. Endpoints devuelven
el modo en cada respuesta para que la UI lo muestre.

### 2. Estructura de datos

```sql
-- Embeddings: BLOB de 384 floats packed con struct
CREATE TABLE signal_embeddings (
  signal_id INTEGER PRIMARY KEY,
  vector_blob BLOB NOT NULL,        -- 384 * 4 bytes = 1.5 KB
  cluster_id INTEGER DEFAULT -1,    -- -1 = noise
  model_version TEXT NOT NULL,
  created_at INTEGER NOT NULL,
  FOREIGN KEY (signal_id) REFERENCES signals(id) ON DELETE CASCADE
);

-- Autonomy: single row
CREATE TABLE autonomy (
  id INTEGER PRIMARY KEY CHECK (id = 1),
  level TEXT NOT NULL DEFAULT 'manual',  -- manual | assisted | autonomous_with_approval
  updated_at INTEGER NOT NULL
);
```

Vectores como BLOB ahorra ~85% vs JSON (`[0.123, 0.456, ...]`).

### 3. Endpoints nuevos

| Endpoint | Propósito |
|---|---|
| `GET /preferences/engine` | Qué modo está activo (real / fallback) |
| `POST /preferences/recluster` | Embed señales sin uno + recluster todas |
| `GET /preferences/clusters` | Lista clusters con sample themes + feedback breakdown |
| `GET /sources/suggestions` | Sugerencias de fuentes basadas en clusters aprobados |
| `GET /autonomy` | Nivel actual |
| `PUT /autonomy` | Cambiar nivel |

### 4. Auto-embed en `_run_scan_internal`

Cada vez que el scanner inserta una señal, automáticamente:
1. Genera embedding del `theme + excerpt + suggested_topic`
2. Lo persiste con `cluster_id=-1` (sin cluster hasta el próximo recluster)
3. Falla silenciosa si hay error (no rompe el scan)

Costo: $0 en fallback. ~30ms por señal en real mode (CPU).

### 5. Scoring de relevancia (sin entrenamiento)

```python
def score_against_feedback(candidate, positives, negatives) -> float:
    pos_avg = mean(cosine_sim(candidate, p) for p in positives)
    neg_avg = mean(cosine_sim(candidate, n) for n in negatives)
    return sigmoid(4 * (pos_avg - neg_avg))  # ∈ [0, 1]
```

Sin ML — pura geometría en el espacio de embeddings. Funciona desde 1
feedback. Reganti Cap 10 §10.3: "lo más simple que funciona".

### 6. UI

`/cazar/fuentes`:
- **Selector de autonomía** prominente al tope con 3 botones:
  - 🛑 Manual (default — sin sugerencias)
  - 🧠 Asistido (sugerencias visibles, founder aprueba)
  - 🤖 Autónomo (añade pendientes de aprobar — futuro)
- **🔁 Re-cluster (gratis)** botón para forzar embedding + clustering
- **🌱 Sugerencias** cards cuando autonomy != manual:
  - Keywords del cluster
  - Botón "+ Crear fuente Bluesky" que pre-llena el form con un Bluesky
    search query usando los keywords

## Why these choices

| Decisión | Por qué |
|---|---|
| Fallback determinista en vez de bloquear sin sentence-transformers | El founder debe poder correr la suite y la UI sin descargar 500MB. Y el fallback es funcional — la pipeline completa works |
| Embeddings dim=384 en ambos modos | Mismo schema cualquier modo. Si cambia de fallback→real más tarde, ejecuta `/recluster` y todo se rehace |
| HDBSCAN con `min_cluster_size=3` | Por debajo crea micro-clusters de ruido. Por encima pierde temas legítimos |
| `cluster_id=-1` para ruido | Convención de HDBSCAN. La UI los oculta |
| BLOB pack vs JSON | 384 floats = 1.5KB BLOB vs ~7KB JSON (`[0.123456789, ...]` con 18 chars/float) |
| Scoring sin entrenamiento | Cold start (1 feedback) y zero ops overhead |
| 3 niveles de autonomía solamente | Cubre 95% de casos. 5+ niveles confunde |
| Suggestions solo de clusters con 👍 ≥1 | Founder no quiere sugerencias de temas que ya descartó |
| Single-row autonomy | App es single-tenant (ADR-017). Multi-tenant en M5 |

## What this does NOT do

- **No entrena un classifier real**: ML supervisado con scikit-learn pipeline
  está documentado pero no implementado. Vendrá cuando el founder tenga
  >100 feedbacks (M5)
- **No auto-añade fuentes**: incluso en `autonomous_with_approval`, el
  founder confirma antes del scan
- **No genera nombres SEO-ish de la fuente**: el botón "Crear fuente"
  pre-llena con `Sugerencia · keyword1 + keyword2` — el founder edita
- **No re-clustering automático**: hay que clickear "🔁 Re-cluster". Un
  cron job (cada 24h) puede agregarse cuando la BD crezca

## Cost model

| Operación | Costo |
|---|---|
| Embedding fallback | $0 (CPU local) |
| Embedding real (CPU) | $0 (sentence-transformers local) |
| Embedding real (GPU si tuvieras) | $0 |
| HDBSCAN sobre N señales | $0 (CPU) |
| Recluster manual (botón) | $0 |
| Sugerencias (TF-IDF keywords) | $0 |

Cero costo LLM. Lo único pago es el LLM de IdeaAnalyzer al click "Analizar".

## Activar embeddings reales en producción

```bash
pip install sentence-transformers hdbscan scikit-learn
# Primera vez: descarga el modelo (~80MB) — automático al primer compute_embedding
```

Tras instalar:
- `GET /preferences/engine` reportará `mode=real`
- `POST /preferences/recluster` re-genera todos los embeddings con el modelo real

## Trigger to revisit

- **>1000 señales y recluster tarda >10s** → mover a job background con APScheduler
- **Founder reporta sugerencias irrelevantes** → calibrar `min_cluster_size` o cambiar a `sentence-transformers` real
- **>100 feedbacks acumulados** → activar scikit-learn classifier real (sklearn
  RandomForest sobre embeddings + feedback como label)
- **Multi-tenant** → tabla pasa a `autonomy(user_email, level, ...)`
