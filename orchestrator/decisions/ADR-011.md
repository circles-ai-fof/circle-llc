# ADR-011: Autonomous Hunter (Sources + Signals + Cron)

**Date:** 2026-05-26
**Status:** Accepted
**Deciders:** Cristian Molina
**Context:** Sprint M3 тАФ circles-ai.ai

---

## Context

The M1 `idea_hunter` only accepts a free-text topic and generates an idea from
the LLM's parametric knowledge. That's the smallest possible hunter тАФ it works
for testing but is **not the hunter the user wanted**.

The desired hunter:
1. Pulls content from **external sources** (RSS, HN, Reddit, Product Hunt, GitHub trending)
2. Distils that content into **structured signals** (theme + score + evidence)
3. Either **manually** (founder reviews + ЁЯСН/ЁЯСО) or **automatically** (cron with
   threshold) turns signals into full workflow runs
4. **Learns** from feedback over time

Reactivating this functionality means promoting `source_scanner` out of
`_deferred/` per R12 (тЙе30 golden cases) тАФ which we **don't yet have**. So this
is a controlled exception, documented here.

## Decision

Add the following components, structured so each can be disabled with one env var:

### 1. `orchestrator/core/source_fetcher.py`

Stdlib-only utility (no `feedparser` / `beautifulsoup4` / `readability-lxml`
deps to keep image size and supply-chain simple):

- `fetch_url(url)` тАФ HTML тЖТ plaintext, with title extraction
- `fetch_rss(url)` тАФ parses RSS 2.0 + Atom feeds
- `fetch_hn_top()` тАФ Hacker News Firebase API (free, no auth)
- `fetch_reddit(subreddit)` тАФ Reddit `.json` endpoint (free, no auth, sometimes 429s)
- `fetch_github_trending()` тАФ scrapes github.com/trending (brittle but free)
- `fetch_product_hunt()` тАФ Product Hunt RSS

**Hard limits** to bound cost + abuse: 8s timeout, 4000 chars/source, 10
items/feed, 5MB max response size.

### 2. `orchestrator/agents/source_scanner.py`

Reactivated from `_deferred/`. Single LLM call per source batch (amortized
~$0.01-0.02 per source). Output: 0-3 `Signal` objects with theme + score +
evidence_urls + suggested_topic. Score floor 0.5 тАФ anything weaker is dropped.

### 3. `orchestrator/core/storage.py` тАФ new tables

```sql
sources (id, kind, target, name, active, last_scanned_at, created_at)
signals (id, source_id, source_kind, theme, score, excerpt,
         evidence_json, suggested_topic, feedback, promoted_run_id,
         created_at)
```

### 4. API endpoints (all auth-required except scan-via-cron)

- `GET    /api/v1/sources` тАФ list catalog
- `POST   /api/v1/sources` тАФ add source
- `DELETE /api/v1/sources/{id}` тАФ remove
- `POST   /api/v1/sources/scan` тАФ manual scan trigger; optional auto-promote
- `GET    /api/v1/signals` тАФ list (with min_score filter)
- `POST   /api/v1/signals/{id}/feedback` тАФ ЁЯСН / ЁЯСО / clear
- `POST   /api/v1/gate/run-from-sources` тАФ workflow seeded by topic OR URLs OR signal_id

### 5. Dashboard pages

- `/cazar/fuentes` тАФ manage source catalog
- `/cazar/senales` тАФ review signals + feedback + promote

### 6. GitHub Actions cron

`.github/workflows/auto-scan.yml` runs every 6h:
1. POST `/api/v1/sources/scan` with `auto_promote_threshold=0.85`
2. Signals scoring тЙе 0.85 are automatically promoted into full workflow runs

Requires two GitHub secrets: `AUTO_SCAN_API_URL` and `AUTO_SCAN_TOKEN`.

## Why NOT bigger / faster / smarter

| Feature | Why not in M3 |
|---|---|
| **X / Twitter API** | $100/mo Basic plan. Free RSS sources cover ~80% of value. |
| **LinkedIn API** | Very restricted, requires partner agreement. Not realistic. |
| **Trend detection over time** | Requires multi-week signal history + NLP clustering. M4+. |
| **Per-source ML scoring (which sources convert)** | Need тЙе100 promoted runs to learn from. M4+. |
| **Authenticated Reddit (no 429s)** | OAuth flow + token refresh. Reddit's free tier 429s after ~30 req/min тАФ fine for 6h cron. |
| **Push notifications on high-score signals** | Email/SMS infra not set up. M4. |

## Cost model

Per scan cycle (1 source):
- 1 source_scanner call тЙИ $0.01-0.02
- Per auto-promoted signal: 1 full workflow тЙИ $0.06

Budget worst case with 5 sources @ 6h cron:
- 4 scans/day ├Ч 5 sources ├Ч $0.02 = $0.40/day source scanning
- Avg 1 auto-promote per cycle: 4 ├Ч $0.06 = $0.24/day workflow
- **~$20/month total LLM cost**

Tighten by raising `auto_promote_threshold` or disabling specific sources.

## Risks + Mitigations

- **Reddit rate-limit**: returns 429 on busy hours. Mitigation: log and continue
  (don't fail the whole scan).
- **Scraping GitHub trending breaks**: HTML structure changes. Mitigation:
  `fetch_github_trending` returns empty silently; logged warning.
- **Token in GitHub Actions secret expires every 7 days** (session TTL). User
  must rotate. M4: convert to long-lived service token.
- **Scanner hallucinates signals from noise**: scoring floor 0.5 + ЁЯСН/ЁЯСО
  feedback loop. M4: train a tiny classifier on labelled signals.

## Trigger to revisit

- > 30 signals promoted to runs тЖТ enough data to start scoring sources by ROI
- < 5 signals/week scoring тЙе0.8 across all configured sources тЖТ scoring is
  miscalibrated; review SourceScannerAgent prompt
- Cron consistently 429s on Reddit тЖТ switch to authenticated OAuth
