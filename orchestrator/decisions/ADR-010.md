# ADR-010: Closed-Beta Email Allowlist Authentication

**Date:** 2026-05-26
**Status:** Accepted
**Deciders:** Cristian Molina
**Context:** Sprint M2 ‚Äî circles-ai.ai

---

## Context

The dashboard at `dashboard.circles-ai.ai` exposes:
- The full `/leads` list with PII (full emails when X-Gate-Secret is provided)
- The `/revision` queue with editable run verdicts
- The "New factory" wizard that triggers `$0.06`-per-call LLM runs

Until now, only X-Gate-Secret (a single shared secret) protected the most
expensive endpoints. Anything else was public-by-default. Before sharing the
URL with co-founders / first beta testers, we need real per-user auth ‚Äî but
*not* a full SMTP/SSO stack, because we have exactly 3 trusted users.

The landing site (`circles-ai.ai`) and per-factory landings (`/f/*`) MUST
stay public ‚Äî they exist to capture leads from paid ads.

## Decision

Email-only **allowlist auth** for the dashboard:

1. **Allowlist** ‚Äî `ALLOWED_EMAILS` env var (comma-separated, lowercased)
2. **Login** ‚Äî POST `/api/v1/auth/login` with `{email}`. If email is in
   allowlist, return a 7-day random token. Otherwise return 403.
3. **Sessions** ‚Äî stored in `sessions` SQLite table (token ‚Üí email + expiry)
4. **Auth attempts log** ‚Äî every attempt (allowed OR rejected) appended to
   `auth_attempts` table with email, IP, user-agent, timestamp. Used for
   audit and later IP-geo lookup.
5. **Dashboard middleware** ‚Äî client-side AuthGuard redirects to `/login`
   when no valid session exists. Token kept in `localStorage`, sent as
   `Authorization: Bearer <token>` header.
6. **Server-side validation** ‚Äî every `authFetch` triggers `/auth/me` to
   catch revoked or expired tokens.

## Why email-only (no password / SMTP)

| Alternative | Why not for M2 |
|---|---|
| Password + bcrypt | Needs password reset flow, password rules, hash storage. 3 users, no need. |
| Magic links (SMTP) | Needs Resend/Postmark/SES setup, deliverability tuning, link expiry handling, secondary auth-state UI. M3 territory. |
| Google/GitHub SSO | Cleaner UX but adds OAuth setup + redirect URIs + dependency. Premature. |
| Single shared password | Worse than email-only: a leak is harder to detect (we can't see "who" logged in). |

For a 3-user closed beta:
- The allowlist itself is the security boundary
- The 3 emails are not public
- Even if leaked, an attacker who guesses the email gets dashboard access ‚Äî same risk as guessing the password in any low-friction system. We accept that risk for beta.
- Migration to magic-links is one new endpoint when we expand to public beta.

## What this protects

- `dashboard.circles-ai.ai/leads`, `/revision`, `/fabricas`, `/agentes` ‚Äî all gated
- API endpoints that require user context (TBD; M3 will require auth on most)
- Lead capture (`POST /api/v1/leads`) ‚Äî still **public**, ads need it
- `GET /api/v1/leads/stats` ‚Äî still **public**, just counts
- `GET /api/v1/leads/{slug}` ‚Äî emails still masked without X-Gate-Secret

## Logging requirement

Every login attempt logs:
- `email` (as submitted)
- `ip` (X-Forwarded-For if present, else direct)
- `user_agent` (truncated to 300 chars)
- `ts` (unix seconds)
- `allowed` (bool)
- `reason` (`ok` | `not_allowlisted` | `malformed_email`)

Use the `GET /api/v1/admin/auth-attempts` endpoint (X-Gate-Secret required)
to retrieve the log. IP geo lookup happens at read time, not at write time
‚Äî keeps the login path fast and bounded.

## Rate limit

Login is rate-limited via the `public_form` tier (5/min, 30/day per IP)
to prevent brute-force enumeration of the allowlist.

## Consequences

**Positive**
- Closed beta usable today, with audit trail
- "Who tried to access?" answered by `auth_attempts` table
- Easy migration: when we open the beta, we just remove the allowlist check; everything else (sessions, attempts log) keeps working

**Negative**
- No password = if one allowlisted email gets phished, attacker can pretend to be that user. Acceptable for beta (small blast radius).
- No 2FA. Acceptable for beta.

## Trigger to revisit

- > 50 attempts/day from suspicious IPs ‚Üí add Turnstile to login form
- Beta opens to public ‚Üí swap to magic-links (SMTP) or OAuth
- A user reports unauthorized access ‚Üí add password + 2FA + session-revoke-all
