# ADR-013: File Upload + Links Bitácora + Pipeline View (M3.2)

**Date:** 2026-05-27
**Status:** Accepted
**Deciders:** Cristian Molina
**Context:** Sprint M3.2 — circles-ai.ai

---

## Context

Three user-requested capabilities after first round of real testing:

1. **File-based source ingestion** — founders accumulate URLs in WhatsApp
   chats, plain-text notes, and Word docs. Manually typing each URL into
   the Fuentes UI is tedious. We need bulk import.

2. **Links bitácora** — every URL the system has ever seen should be
   auditable: where it came from, when, whether it was analyzed, and what
   the LLM concluded. Sources can come and go, but the audit trail stays.

3. **Pipeline view** — the founder needs to see at-a-glance which workflow
   runs are in each phase (pending_review / iterate / pass / kill / overridden)
   without clicking through individual run records.

## Decision

### 1. File upload endpoint

```
POST /api/v1/sources/import-file   (multipart/form-data, auth required)
```

Accepts `.txt`, `.csv`, and `.docx`. Max 5 MB. Pipeline:

- `file_parser.parse_file(name, bytes)` dispatches by extension:
  - `.docx` → `python-docx` library (optional dep; degrades gracefully)
  - everything else → UTF-8 decode with latin-1 fallback
- `file_parser.extract_urls(text)` runs a conservative regex and dedup
- For each URL:
  - Logged in `links_log` table with `source_file = <filename>`,
    `status = 'pending'`
  - Added as `url`-kind source (skipped if already present)

WhatsApp chat exports work as plain `.txt` (the export format is just text
with `[date time] Name: message` lines).

### 2. Links bitácora — new table + LLM categorizer

New table:
```sql
links_log (id, url, source_file, status, idea_summary, sector, area,
           rejection_reason, created_at, analyzed_at)
```

Status state machine: `pending → analyzed | rejected | error`.

New agent `link_analyzer` reactivated from `_deferred/`. Single LLM call
per URL (cost ~$0.005-0.01). Output:

```jsonc
// status="analyzed"
{ "status": "analyzed", "idea_summary": "...", "sector": "saas", "area": "..." }

// status="rejected"
{ "status": "rejected", "rejection_reason": "Página promocional sin pain point claro" }
```

Endpoints:
- `GET /api/v1/links?status_filter=...` — list with counts by status
- `POST /api/v1/links/analyze` — batch-analyze pending links
  (bounded by `max_to_analyze`, default 10, max 50)

### 3. Pipeline view

```
GET /api/v1/pipeline
```

Buckets every run from `runs_store.values()` into 5 columns:

| Column | Criterion |
|---|---|
| `pending_review` | `needs_human_review=True` AND no override |
| `iterate`        | verdict=iterate (no override, no review needed) |
| `pass`           | verdict=pass (no override) |
| `kill`           | verdict=kill (no override) |
| `overridden`     | `human_override` is not null |

Dashboard page at `/pipeline` auto-refreshes every 8s. Each card shows
idea title, confidence, cost, override badge, and a deeplink to the
landing slug.

## Why these choices

| Decision | Why |
|---|---|
| Stdlib regex for URL extraction | No `validators`, `rfc3987`, or full URL parser dep. Trailing punctuation trimmed in post-processing. |
| `python-docx` as optional | DOCX is rare in B2B founder chats. If user uploads one and lib is missing, return helpful 400 instead of installing 50 transitive deps. |
| 5 MB file size cap | A WhatsApp chat export is ~100 KB - 2 MB. 5 MB covers 95% of cases without exposing the server to memory abuse. |
| Two-step ingestion (extract → analyze separately) | File upload is sync (<5s). LLM analysis is async per click — bounds cost and lets the founder review before spending. |
| Pipeline as 5 fixed columns | Phases are static, not configurable. Reganti Cap 11 §11.3: avoid premature flexibility — when we have 50 verdicts, then think about adding columns. |
| Auto-refresh every 8s | Sensible-but-not-aggressive polling. Cheap enough that we don't need SSE/WebSockets for M3. |

## What this does NOT do

- **No real-time push** (SSE/WebSocket). Auto-refresh is cheap polling.
  Upgrade in M4 when we have ≥5 simultaneous users.
- **No PDF / HTML upload**. PDF would need PyMuPDF (~30 MB binary).
  Defer until requested. HTML is just a URL — paste the URL into Fuentes.
- **No automatic analysis of imported URLs**. We separate ingestion (free,
  fast) from analysis (LLM, $0.005-0.01 each, batched by the user).
- **No source-quality feedback loop on link_analyzer**. The bitácora is
  manual review for now. ML for auto-rejection comes when we have ≥500
  analyzed links.

## Cost model

- File upload: $0 (stdlib only, no LLM)
- Per link analysis: ~$0.005-0.01 (single Claude call, ~600 tokens)
- 100 links from a chat = ~$0.50-1.00 if you analyze all
- 5 MB file = ~50K characters = ~10K tokens of prompt if everything went
  to LLM, but we DON'T do that — we extract URLs first (free) and
  analyze them one by one with bounded prompt size.

## Tests

- `test_file_parser.py` (10): URL extraction, dedup, trailing punctuation,
  WhatsApp chat format, UTF-8 + latin-1, DOCX graceful fallback
- `test_link_analyzer.py` (4): mock mode rejection/acceptance, dataclass
- `test_pipeline_and_links.py` (12): pipeline buckets, links auth/filter/
  pagination, file import happy/dedup/disallowed-ext/empty/no-auth

387 tests passing (was 360).

## Trigger to revisit

- > 100 imported URLs / week → make analysis automatic with budget cap
- < 10 imported URLs / week → roll back; was over-built
- Founders ask for PDF → add PyMuPDF in a focused follow-up
- Pipeline column counts skewed (>80% in one column) → reconsider phases
